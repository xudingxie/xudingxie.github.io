CIP通讯个人学习总结

简介

属性

CIP：点对点面向对象协议，处于应用层中。
它可以通过使用三种上层网络协议。

  • DeviceNet
  • ControlNet
  • EtherNet/IP

目的

CIP协议的目的是连接PLC，开发PLC通讯组件。

实现的功能

  • 工业的对象的标注定义
  • 实现通信

报文类型

  • 隐式报文
    隐式报文使用生产者/消费者（支持多播）主要用于传输实时性要求较高的数据，例如I/O报文
    都是已连接
    通讯类型I/O数据交换
  • 显式报文
    源/目的地 同时Profibus也是源/目的地（不支持多播，利用IP指定某一个对象）
    显式报文又可以分为已连接传输与未连接传输
    通讯类型 请求/响应 交换

地址

  • 公开的
  • 供货商指定的
  • 对象指定的

连接对象

  • 通过连接对象创建连接
  • 通过连接管理器对象创建链接

通过连接对象创建连接

当⼦⽹定义通过Connection Object创建连接时，CIP设备应⽀持该类的Create服务。 Create 服务使⽤类定
义的默认属性值实例化连接实例。连接实例是通过对每个连接实例属性的单独访问来配置的。需要单独的服务
请求（Apply_Attributes，服务代码 0x0D）将连接转换为 Established 状态。

通过连接管理器对象创建链接

当⼦⽹定义通过Connection Manager Object创建连接时，CIP设备应⽀持该类的Forward Open服务。成功时，
Connection Manager 实例化 Connection 类的⼀个实例。此连接实例配置有在 Forward Open 服务中发送的
值，并转换为已建⽴状态。此单个 CIP 服务请求在内部建模为单个连接类服务请求（使⽤创建服务）和多个内部服务
请求（使⽤ Set_Attribute_Single 和 Apply_Attributes 服务）。⽀持通过连接管理器创建连接的设备可能会也
可能不会提供对连接类实例的外部可⻅性。

报文格式（应该是与欧姆龙通讯的格式）

通讯首先是

  1. 注册信息
  2. （optional）fordword_open/forword_large_open
  3. 相关通讯
  4. (optional) forword_close
  5. 解除注册

Header+CommandSpecificData+CIP消息
Ethernet/IP 是一个应用层标准，它将网络中的所有设备都视为一系列的对
象。
在应用层，CIP 按照对象来封装数据，包括 3 类对象：
通用对象：连接对象、报文路由对象、身份对象。
网络特定对象：TCP/IP 接口对象、ControlNet 接口对象、DeviceNet 接口对象。
应用对象：和具体的应用和设备相关，比如有 AD/DC 变频器、离散量、模拟量 I/O 对象等。
其中寻找地址中比较重要的有四个属性

  • 设备（MAC id）
  • class id
  • instance id
  • attrbute id/ServiceID
    类ID分为两个部分，公共对象（范围：0x0000–0x0063, 0x00F0–0x02FF），厂家自定义对象（范围：0x0064–0x00C7, 0X0300-0X04FF）。其它范围为预留部分。
    实例ID也分为两个部分，公共实例（范围：0x0001–0x0063,0x00C8-0x02FF），厂家自定义实例（范围：0x0064-0xxC7,0x0300-0x04FF）。其它范围为预留部分。
    属性ID,公共属性（范围：0x0000–0x0063，0x0100–0x02FF，0x0500–0x08FF），厂家自定义属性（范围：0x0064–0x00C7，0x0300–0x04FF，0x000–0x0CFF）
    class可以理解成为某一类：human
    instance实例，就是某一个体：老王
    attribute可以理解成为对外暴露的属性。:age,name
    多字节整数部分采用小端传输，标准Internet网络协议是big-endian
